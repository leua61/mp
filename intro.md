# CODEX v2.4 使用说明与课程生成规范

本文件同时作为项目使用说明、目录约定和课程生成规范。项目中不再单独保留 `README.txt`。

## 一、快速使用
打开 `index.html` 使用。测试时可打开 `perfect.html`，进入全关卡 5/5 的完美测试视图。

不要单独移动 `index.html` 或 `perfect.html`，必须和 `m.html`、`b.css`、`lessons/`、`cards/` 文件夹放在一起。

## 二、项目定位
本项目为本地 HTML 教学训练网页，采用「主框架 + 课程 DLC」分离架构。

- 主框架统一负责通用交互与界面渲染
- 课程文件仅提供题目内容数据，无需关心界面实现逻辑
- 卡片绑定信息统一写在 `lessons/card_bindings.js`，不要写进各课程 JS；卡牌图片放在顶层 `cards/<课程id>/` 文件夹。

## 四、核心分工规则
### 4.1 五线谱容器（全局通用组件）
- 是页面通用内容承载容器，并非题目专用组件
- 可承载文字、选择题、解析、动画、canvas、svg、轻量互动等各类内容
- 课程文件禁止复制容器代码，仅提供纯内容数据
- 容器渲染、样式、交互逻辑由主框架统一实现

### 4.2 主框架职责
课程切换、五线谱容器渲染、随机抽题、开始按钮、计时器、作答锁滚动、答完解锁、掌握等级判定、选项正误判断、扩展区展示、内容超出自适应拉长、背景与装饰动画。

### 4.3 课程文件职责
仅通过 `Game.addLesson()` 接口注册课程数据，不处理任何界面、样式、交互逻辑。

## 五、课程数据格式规范
### 5.1 顶层结构
```js
Game.addLesson({
  id: '课程唯一标识（英文小写+下划线）',
  title: '课程显示名称（中文）',
  topics: [ /* 知识点数组 */ ]
});
```

### 5.2 知识点节点
每个知识点对象对应一个训练卡，包含元信息与题目生成函数：
```js
{
  id: '知识点唯一ID',
  title: '知识点标题（建议前缀标注类型：纯基础/计算/随机等）',
  makeQuestion() {
    return {
      stem: '题干文本',
      answer: '正确答案文本',
      options: ['选项1', '选项2', '选项3', '选项4'],
      hint: '基础知识提示（必填）',
      solution: '本题解法解析（可选）'
    };
  }
}
```

> 主框架仍兼容旧字段：`t`、`ts`、`mk`、`q`、`a`、`pool`，但新增内容建议全部使用完整字段名。

### 5.3 可递归路线树 / 同卡子级别
当同一题卡内存在多个必须经历的小题，或某个小题内部仍存在必须覆盖的固定分支时，统一使用 `subs` 描述下级节点。`subs` 可以递归嵌套，设计上允许无限层级；实际课程中通常使用到「题卡 → 小题们 → 小题内部分支」三级。

```js
{
  id: 'fraction_recursive_route',
  title: '07｜分数转化｜基础百化分、9/11互逆与七分数',
  type: '同卡子级别',
  levelType: 'sequence',
  unlockMode: 'sequence',
  routeSchema: 'recursive_route_tree_v1',
  progressScope: 'card_children',
  progressLabel: '题卡掌握',
  localProgressLabel: '局部掌握',
  subs: [
    {id:'fraction_basic_percent', title:'25｜基础分数转化', makeQuestion(){/* 叶子题 */}},
    {
      id:'fraction_9_11_inverse',
      title:'27｜9 和 11 骨架互逆',
      progressScope:'node_children',
      progressLabel:'局部掌握',
      subs:[
        {id:'divide_by_11', title:'27-1｜除以 11 → 乘 9 骨架', makeQuestion(){/* 叶子题 */}},
        {id:'multiply_by_9', title:'27-2｜乘 9 骨架', makeQuestion(){/* 叶子题 */}}
      ]
    }
  ]
}
```

- 顶层题卡右上角的 `掌握 x/y` 表示：当前题卡直属小题通关数 / 直属小题总数。
- UI 渲染采用“同构平铺”原则：每一层都是同款路线按钮区 + 本层进度条；当进入有下级的节点时，不在按钮内部嵌套小卡片，而是在下方追加下一层同款路线板块。
- 每个有 `subs` 的节点都有自己的本层进度条，例如进入 `27｜9 和 11 骨架互逆` 后，下方会新增第二层路线按钮区与 `掌握 2/4` 进度条，表示该小题直属分支通关数 / 直属分支总数。
- 叶子节点没有下级 `subs` 时，继续使用自身 `5/5` 熟练度。
- 每个有 `subs` 的节点都可以继续设置 `levelType` / `unlockMode`，解锁规则只作用于该节点的直属子级。
- `subs` 只表示“本节点有下级分支”，不自动等于顺序解锁；是否需要按顺序通关，必须由该节点自己的 `unlockMode` 独立决定。
- 简单认知、考情、规则问答、驿站、理念类关卡可以保留二级分支，但本层应使用默认 `all`，学生可任选入口通关。
- 为兼容旧框架，含下级的节点也可以保留一个兜底 `makeQuestion()`；当前主框架会优先渲染其下级路线。


### 5.4 分支关卡解锁与随机策略
同卡子级别题卡可设置 `levelType` / `unlockMode`：

```js
{
  id: 'skill_card',
  title: '技能卡',
  type: '同卡子级别',
  levelType: 'sequence',
  unlockMode: 'sequence',
  subs: [ /* 子关卡 */ ]
}
```

- 不写时默认为 `all`：所有子关卡默认全解锁，适合基础认知、简单规则、考情结构、驿站、理念说明等不需要递进门槛的卡片。
- 写 `sequence` 时为按顺序通关解锁：第 1 个子关卡默认开放，上一子关卡掌握度达到 `5/5` 后解锁下一子关卡；仅适合确实有学习先后关系的分数转化链、误差两步法、划线法方向训练等强递进训练。
- 主框架统一只识别 `sequence` 作为顺序解锁标记；其他写法按默认全解锁处理。
- 已通关子关卡入口会自动增加一层细金光描边，作为 `5/5` 通关标识。

随机策略建议显式区分两类：

```js
{ randomMode:'coverage',  randomLabel:'路线型随机' }   // 必须覆盖固定分支
{ randomMode:'parameter', randomLabel:'参数型随机' }   // 同一题型只随机数值
```

- `coverage` 表示路线型随机，也就是训练语义上的“假随机”：目标是保证固定分支都被经历、记录、掌握，不能只靠普通随机碰到。
- `parameter` 表示参数型随机，也就是训练语义上的“真随机”：题型结构不变，只替换数字、单位、位置等参数；不要求遍历全部可能值，只要求练到熟练。
- 判断原则：如果某个变化代表不同考点，就做成 `subs` 路线分支；如果只是同一考点换数值，就留在 `makeQuestion()` 内做参数随机。

### 5.5 题卡标签与驿站规则
课程知识点可通过 `tags`、`type`、标题前缀表达题卡性质。为避免同义标签失控，题卡主类型统一收敛为四类：

- `基础`：考情、概念、规则、常识性识别等基础认知内容。
- `技巧`：需要步骤、推导、计算、误差处理、方法选择的训练内容。
- `运用`：结合选项、实战策略、综合判断、训练理念、场景迁移的内容。
- `驿站`：休息、翻译、彩蛋、补充说明、轻量解释等非主线内容。

补充规则：

- 每个题卡或同卡子级别建议至少包含一个主类型标签，可继续叠加细分标签，例如 `['技巧','误差速算','划线法']`。
- 细分标签用于检索、筛选和说明，不替代四类主类型。
- `驿站` 不受章节主线和知识线顺序限制，可按节奏需要插入 `topics` 任意位置。
- 全局检查课程结构时，不把 `驿站` 的插入位置视为组织混乱、知识点缺漏或顺序错误。
- `驿站` 可使用普通题卡、同卡子级别或分支入口，是否顺序解锁由具体互动目的决定。


### 5.6 卡片绑定与图片位置
卡片信息不写在课程 JS 内。课程文件只注册 `topics`，卡片归属、展示顺序、图片路径统一写在 `lessons/card_bindings.js`：

```js
Game.addCardBindings({
  data_analysis_speed_01: [
    { id:'card_id', image:'cards/data_analysis_speed_01/card.png' }
  ]
});
```

卡牌图片按课程分文件夹存放：`cards/<课程id>/`。这样 cards 不再打进单个 lesson 目录，移动或新增课程时，题目内容、卡牌图片位置、全局绑定表三者保持分离。


### 5.7 完美测试存档入口
打开 `perfect.html` 会直接进入完美测试视图；也可用 `m.html?perfect=1`。该模式只改读取机制：所有关卡与子关卡在页面内按 `5/5` 显示与解锁，但不会写入或覆盖 `localStorage` 中的真实存档，退出后从普通 `index.html` 进入仍读取原存档。

- 用于测试全解锁、已通关金边、课程切换等状态。
- 该入口不写入本地浏览器 `localStorage`，普通入口再次打开时仍看到真实存档。
- 存档位置为本页 `localStorage` 的 `sf2` 与 `sf2c`。

### 5.8 顶部功能区与设置页
主界面顶部保留一个独立功能区，专门放置全局功能按钮。

- 每个功能按钮支持鼠标悬浮小提示。
- 点击按钮打开对应的大页面弹层。
- 当前内置「设置」按钮。
- 设置页目前只提供存档导入、存档导出功能。
- 导出内容包含 `sf2`（关卡掌握度/通关进度）与 `sf2c`（当前课程）。
- 导入存档后会刷新页面，重新读取新的本地存档。

## 六、扩展区显示规则
扩展区位于题目下方，含「基础知识提示」「本题解法」两栏，严格遵循以下规则：

1. 存在 `hint` 字段，才显示「基础知识提示」
2. 存在 `solution` 字段，才显示「本题解法」
3. `hint` 为必填字段，所有题目都应提供
4. 纯基础知识题不写 `solution`；基础知识本身即为解法时，仅保留 `hint`
5. 题目对象缺少 `hint` 视为不合规

## 七、分节与分隔行
### 7.1 标准写法
- 仅写在 `topics` 数组内，和题目对象平级
- 字段固定 `k:'s'`
- 只保留 `x` 作为分隔栏显示文字
- 不写 `makeQuestion`
- 分隔对象末尾必须加逗号
- 禁止给分隔对象增加 `id`、`tags` 等题目字段

### 7.2 示例
```js
const topics = [
  {k:'s', x:'第一章：资料分析考情与题型结构'},
  {id:'material_question_count', title:'纯基础｜材料题量', makeQuestion(){
    return makeQuestion('一篇资料材料配套选择题数量？','5道',['3道','4道','5道','8道'],'常规考法一篇配 5 题');
  }},
  {k:'s', x:'第二章：三不看速算原则'},
  {id:'placeholder_definition', title:'纯基础｜占位符定义', makeQuestion(){
    return makeQuestion('0/小数点/百分号统称？','占位符',['有效数字','占位符','单位','误差值'],'三不看仅忽略占位符');
  }}
];
```

## 八、代码风格要求
- 不写冗余注释，不写多余兜底逻辑
- 课程文件只写数据和题目生成逻辑
- 新增命名优先使用完整英文单词，不使用过短缩写

1. 将课程 JS 文件放入 `lessons/` 目录
2. 在根目录 `catalog.js` 中新增对应课程路径
3. 课程文件通过 `Game.addLesson()` 注册
4. 禁止单独移动 `index.html`，必须与 `m.html`、`b.css`、`lessons/` 保持同层级

zip 包根目录直接包含：`index.html`、`m.html`、`perfect.html`、`b.css`、`catalog.js`、`CODEX.md`、`lessons/`。


## 附：卡牌专区

- 主框架功能区提供「卡牌专区」入口，用于查看当前课程已收集卡牌。
- 普通模式读取真实 `localStorage` 卡牌存档；默认没有卡牌，不会自动写入。
- 完美测试模式只在页面内展示当前课程配置的全部卡牌，仍不写入或覆盖真实存档。
- 课程可在 `Game.addLesson()` 顶层提供 `cards` 数组；本次 `data_analysis_speed_01` 配置 3 张角色卡。


### 递归路线 UI 补充：上级节点进度与最近位置

- 平铺路线中，每一级仍复用同一个路线板块：路线图 + 本级进度条。
- 如果某个按钮代表“有下级的节点”，按钮标题后方以内联普通文本显示该节点的子级完成比重，例如：`27｜9 和 11 骨架互逆 4/4`。这样不进入该节点，也能看到它内部是否完成，并且能直观看出它还有下级路线。
- 下级路线的显示编号使用当前层级的局部顺序号：`1｜...`、`2｜...`、`3｜...`，不继承父级编号前缀，例如不显示 `27-1`。
- 本地存档额外记录 `sf2ui`：最近打开课程、每个课程的滚动位置、每个题卡最近选中的递归路线路径。该记录只保存界面位置，不改变题目掌握度。


### 路线节点进度标签显示规则

父级路线按钮若拥有下级节点，可在按钮内显示该子树的完成比重，但只显示 `X/Y`，不附加“掌握”或“按顺序通关解锁”等额外文字。顺序解锁仍由数据字段控制，但不在按钮进度标签中重复展示。

- 每一级路线板块的总进度保持同构显示，统一写作 `掌握 X/Y`；只有路线按钮内的子级进度尾标只写 `X/Y`。

### 分文件课程结构补充

课程支持两种注册方式并存：

1. 旧的大文件格式：单个课程 JS 直接调用 `Game.addLesson({id,title,topics})`，占位符课程继续使用该方式。
2. 新的分文件格式：在 `lessons/<课程id>/` 课程文件夹内直接放每个单题卡 JS，不再创建 `cards/` 子文件夹，也不再套一层同名文件夹。

`data_analysis_speed_01` 当前采用第二种结构：

```text
lessons/data_analysis_speed_01/
  _common.js
  00_section_exam.js
  01_exam_material_structure.js
  ...
  24_supplement_training_mindset.js
  _finish.js
```

其中 `_common.js` 建立题库公共函数与课程暂存区，单题卡文件逐个追加题卡，`_finish.js` 最后将该文件夹内容注册为同一节课。运行时仍然把它识别为一节完整课程。

### 01 课程拆分与下级路线更新记录

- `data_analysis_speed_01` 以带有 `驿站｜翻译` 的旧版 JS 为基底重新拆分，驿站题卡保留在三不看模块中。
- 课程文件夹仍为 `lessons/data_analysis_speed_01/`，内部直接放单题卡 JS；不使用 `cards/` 子文件夹。
- 能明确合并为固定分支路线的同主题小题已改为父题卡 + 下级路线形式。
- 子级标题尽量去掉重复主题词，例如三不看下级使用 `实战`、`百分号实战`、`适用范围`、`底层逻辑`。
