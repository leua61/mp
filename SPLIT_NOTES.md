# 拆分说明

本版本只做低风险拆分：不改原函数体、不改课程数据接口、不改存档键。

## 文件职责

- `m.html`：主页面薄壳，只负责加载样式和脚本。
- `css/main.css`：原 `m.html` 内联主界面样式。
- `b.css`：原背景样式，继续独立保留。
- `js/noanim.js`：读取 URL 参数，决定是否禁用动画。
- `js/state.js`：全局状态、存档键、基础工具、SVG/容器片段。
- `js/utils.js`：选项生成、文字规范化、尺寸自适应、滚动锁定、解锁规则。
- `js/templates.js`：题卡、分隔栏、课程切换、功能区 HTML 模板。
- `js/cards.js`：卡牌专区逻辑。
- `js/settings.js`：设置页、存档导入导出逻辑。
- `js/quiz.js`：开始作答、判题、解析区、掌握度更新。
- `js/lessons-runtime.js`：课程切换、页面渲染、`Game.addLesson()` / `Game.loadLessons()`。
- `js/boot.js`：背景动画、计时器、resize 监听、加载根目录 `catalog.js`。

## 使用方式

把这些文件放回项目根目录，并保留原来的 `lessons/` 文件夹结构。入口仍然打开 `index.html`。

## 注意

`js/*.js` 的加载顺序不能随意调换；目前顺序写在 `m.html` 底部。

## 自动题号

题卡标题序号由主框架自动生成，不写入课程文件。

- 位置：`js/lessons-runtime.js` 的 `rd()` 按当前课程中非分隔栏题卡顺序生成 `_n`。
- 显示：`js/templates.js` 的 `title(c)` 在渲染标题时拼成 `序号｜原标题`。
- 分隔栏不参与计数：标准 `{k:'s'}`，以及兼容 `k:'section' / k:'sep' / type:'section' / section:true / isSection:true` 等旧写法。
- 课程文件仍只维护原始 `title/t`。


## 2026-06-27 内部子关卡编号修正

- 单张题卡内部的 `subs` 会按可训练子关卡自动生成连续编号。
- `subs` 中的章节/分隔项不再渲染为子关卡按钮，也不参与顺序解锁和完美测试进度填充。
- 子关卡按钮仍使用原始 `subs` 下标作为 `data-sub`，避免破坏已有存档 key。


## global-number-fix
- 内部小题按钮只显示全局题号：`12｜找数据能力`。
- 移除此前误加的本卡内序号层，不再出现 `1｜12｜...`。
- `subs` 内的章节/分隔项不占全局题号，也不渲染为小题按钮。


## 线性小关卡箭头

- 主程序统一只识别 `sequence` 类型的小关卡；其他写法按默认全解锁处理。
- 只有线性解锁的同卡小关卡，才会在真实小关卡按钮之间自动插入 `⮕`。
- 章节/分隔项不参与小关卡渲染，也不会产生箭头。
- 课程文件不需要手写箭头。
