# CODEX

本项目是本地 HTML 教学训练网页。

## 结构

index.html 是主框架。
lessons/catalog.js 是课程入口。
lessons/*.js 是 DLC 课程文件。

## 五线谱容器

五线谱容器是全局通用容器。
它不是题目专用组件。
它可以承载文字、选择题、解析、动画、canvas、svg、小互动。
课程文件禁止复制五线谱容器代码。
课程文件只提供内容。
主框架统一负责容器渲染。

## 主框架职责

课程切换、五线谱容器、随机选择题、开始按钮、计时器、作答锁滚动、答完解锁、掌握等级、选项判定、扩展区、内容超出动画拉长、容器高度按内容贴合、背景和装饰动画。

## 课程文件职责

课程文件只注册课程数据。

格式：

Game.addLesson({
  id:'course_id',
  title:'课程名',
  topics:[{
    id:'topic_id',
    title:'知识点名',
    makeQuestion(){
      return {
        q:'题干',
        a:'答案',
        pool:['选项1','选项2','选项3'],
        hint:'基础知识提示',
        solution:'本题解法'
      }
    }
  }]
})

## 扩展区规则

有 hint 才显示“基础知识提示”。
有 solution 才显示“本题解法”。
纯基础知识题不要写 solution。
基础知识本身就是解法时，只保留 hint。
两个字段都没有，不显示扩展区。

## 代码风格

不要写注释。
不要写多余兜底。
不要写检验流程。
保持模块化。
函数不要太长。
命名短但要有语义。
能合并的短语句适当合并。
不要为了合并破坏可读性。
主框架优先省空间、省耗电。
容器底部留白应接近顶部内间距，不要保留大块空尾巴。

## 打包

zip 根目录直接包含：
index.html
README.txt
CODEX.md
lessons/

禁止多包一层外部文件夹。
